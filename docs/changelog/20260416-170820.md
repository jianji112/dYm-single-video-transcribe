# 添加用户时可选下载作品

## 任务描述
新增系统设置开关 `download_post_on_add_user`（默认开启）。当用户在添加用户时传入"作品链接"，若开关开启：
- 用户不存在 → 添加用户 + 后台下载该作品
- 用户已存在 → 不再抛"用户已存在"，而是检查该作品是否已下载，未下载则后台下载
- 已下载 → 不重复下载，toast 提示
- 用户主页链接或开关关闭 → 保持现有行为

下载为**异步后台**执行，不阻塞 `user:add` IPC 返回。完成/失败通过新频道 `user:addPostProgress` 推送给渲染进程，由 `AppLayout` 统一展示 toast。

## 最终执行结果
- 新增 setting key: `download_post_on_add_user`，默认 `'true'`
- `user:add` IPC 返回 `{ user: DbUser, postDownload: { status, awemeId? } }`；status 枚举：`downloading | already-downloaded | disabled | unavailable | not-video-link`
- 新增 downloader 导出 `downloadSinglePost(user, awemeData)`，复用现有 `DouyinDownloader` + `createPost` + `validateDownloadFolder` 流程
- 新增广播频道 `user:addPostProgress`（payload `{ awemeId, nickname, status: 'success' | 'failed', error? }`）
- 渲染进程：`AppLayout` 作为单一订阅点，按状态推送 toast；`UsersPage` / `AppLayout` 的添加 toast 文案按 postDownload.status 差异化
