# 2026-05-06 — Live 图视频播放 + 文件管理封面修复

## 待办
- [x] 修复文件管理页超过 50 项后旧封面变空白
- [x] 让 Live 图（图集类型 68）下载的 mp4 在桌面端代替静态图播放
- [x] Web feed 同步支持 Live 图 mp4 播放
- [x] 写入 changelog

## 问题与根因

### 1. 文件管理页 50+ 后封面空白
`FilesPage.tsx#loadCoverPaths` 每次加载下一页时执行 `setCoverPaths(paths)` —— 直接把整个 state 替换为新页 50 个封面，前面已加载的 key 全部丢失，UI 渲染回退到空白封面。

### 2. Live 图无法播放 mp4
`media.ts#findMediaFiles` 在 `aweme_type === 68` 分支只过滤 `webp/jpg/jpeg/png`，文件夹中实际存在的同名 mp4 被忽略，前端只能拿到静态图。

## 实施

### `src/renderer/src/pages/settings/FilesPage.tsx`
将 `setCoverPaths(paths)` 改为合并：`setCoverPaths((prev) => ({ ...prev, ...paths }))`。

### `src/main/services/media.ts`
- `MediaFiles` 新增可选字段 `imageVideos: (string | null)[]`
- 图集分支扫描 mp4，按 basename 与 image 配对；basename 全不匹配且数量相等时按顺序兜底配对

### `src/main/services/web-browser.ts`
`/api/posts` 响应的 `media` 增加 `imageVideoUrls: (string|null)[]`。

### `src/renderer/src/components/MediaViewer.tsx`
图集渲染时，若当前 index 有对应 mp4，使用 `<video autoPlay loop muted playsInline poster={image}>`；否则保持 `<img>`。

### `resources/web/app.js`
- 图集 stack 中：有 mp4 的 slot 渲染 `<video class="js-gallery-video">`，无则保留 `<img>`
- `updateImageStory`：切换时对当前 video 调 `play()`、对其他 video 调 `pause()`
- `activateStory`：图集激活时启动首张 video
- `pauseAll`：暂停 `.js-gallery-video`

## 影响面
- 5 个文件，新增字段全部可选，向后兼容旧调用方与旧响应消费者。
- 不涉及 dy-downloader 上游库改动。
