# Fix: 下载 0KB/不完整文件问题 (Issue #3)

**日期**: 2026-03-10
**类型**: Bug Fix
**Issue**: #3

## 任务描述

修复下载作品时出现 0KB 空文件和不完整视频文件的问题，并增加重新下载功能。

## 根因分析

1. dy-downloader 库下载失败后不清理残留文件（0KB 或半写文件留在磁盘）
2. dy-downloader 的 existsSync 跳过逻辑不检查文件大小，0KB 文件被视为已完成
3. downloader.ts/syncer.ts 在 createDownloadTasks 后无条件入库，失败的下载也被记录
4. 无文件完整性校验（Content-Length、文件大小等）
5. 坏记录入库后，后续运行通过 getPostByAwemeId 跳过，永远无法修复

## 待办

- [x] Phase 1: 修复 dy-downloader 上游库（临时文件策略 + 文件大小校验 + 失败清理）
- [x] Phase 2: downloader.ts / syncer.ts 增加下载后校验层
- [x] Phase 3: 数据库层新增 deletePostByAwemeId、scanBrokenPosts 方法
- [x] Phase 4: 新增 IPC 接口 + 前端重新下载 / 扫描修复 UI
- [ ] Phase 5: 打磨（进度展示优化）

## 执行结果

### Phase 1: dy-downloader 修复
- `downloadFile` 方法改为先写入 `.tmp` 文件，完成后 rename 为最终文件名
- 下载完成后校验文件大小，0KB 文件视为失败
- `existsSync` 跳过逻辑增加文件大小检查，0KB 文件重新下载
- catch 块中清理 `.tmp` 残留文件

### Phase 2: 校验层
- 新增 `download-validator.ts`，提供 `validateDownloadFolder`、`cleanupFailedDownload`、`checkPostFileIntegrity`
- `downloader.ts` 和 `syncer.ts` 在 `createDownloadTasks` 后调用校验，失败自动重试 1 次
- 校验不通过不入库，避免坏记录锁死

### Phase 3: 数据库
- 新增 `deletePostByAwemeId` — 按 aweme_id 删除记录
- 新增 `getPostsByUserIdAll` — 获取用户全部作品（无分页）

### Phase 4: IPC + UI
- 新增 `post:scanBroken` — 扫描所有损坏文件
- 新增 `post:redownload` — 删除记录+清理文件，等待下次同步重下
- 新增 `post:batchRedownload` — 批量重下
- HomePage 和 FilesPage 右键菜单增加"重新下载"选项
